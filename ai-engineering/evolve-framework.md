## Overview

An integrated framework combining SuperClaude behavioral modes, CCPM project management, and Claude Flow orchestration to enable truly autonomous, systematic software development at scale.

**Problem Solved:** Traditional AI-assisted development suffers from lost context, inconsistent quality, and ad-hoc methodologies. Evolve provides systematic coordination of 54+ AI agents with perfect context retention and proven quality gates.

## SPARC Methodology

Systematic 5-phase development workflow transforming requirements into production code:

1. **Specification** → Requirements analysis with stakeholder dialogue
2. **Pseudocode** → Algorithm design before implementation
3. **Architecture** → System design with pattern validation
4. **Refinement** → Test-driven implementation with quality gates
5. **Completion** → Integration testing and deployment validation

### 54+ Specialized Agents

**Core Development:**
- coder, reviewer, tester, planner, researcher

**Swarm Coordination:**
- Hierarchical, mesh, adaptive topologies
- Byzantine fault tolerance for reliable consensus
- Cross-session memory persistence

**Domain Specialists:**
- backend, frontend, ML, DevOps, security, API design
- GitHub integration (PR management, code review automation)
- Performance optimization (32.3% token reduction)

### Coordination Topologies

```python
# Hierarchical - Top-down task decomposition
swarm_init(topology="hierarchical", maxAgents=8)

# Mesh - Peer-to-peer collaboration
swarm_init(topology="mesh", maxAgents=6)

# Adaptive - Dynamic topology switching
swarm_init(topology="adaptive", strategy="auto")
```

## CCPM Project Management

Spec-driven development with GitHub synchronization:

### PRD System
- Brainstorming → structured requirements
- Automated epic decomposition
- GitHub issue generation

### Issue Workflow
```bash
# Create product requirement
/pm:prd-new "feature description"

# Decompose into tasks and sync to GitHub
/pm:epic-oneshot

# Start implementation with specialized agents
/pm:issue-start <issue-number>
```

### Privacy Protection
- Path sanitization (no absolute paths in public docs)
- Repository validation (prevents accidental template pollution)
- Portable documentation (works across development environments)

## Technical Implementation

### Installation (< 5 minutes)

```bash
# 1. Clone repository
git clone https://github.com/kvnloo/evolve.git
cd evolve

# 2. Install Claude Flow (required)
claude mcp add claude-flow npx claude-flow@alpha mcp start

# 3. Verify setup
npx claude-flow sparc modes  # Should list 5 SPARC phases

# 4. Optional: Enhanced coordination
claude mcp add ruv-swarm npx ruv-swarm mcp start
```

### First Feature Development

```bash
# Create PRD through guided brainstorming
/pm:prd-new "user authentication system"

# Decompose into tasks and sync to GitHub
/pm:epic-oneshot

# Start implementation with SPARC methodology
/pm:issue-start <issue-number>

# Automatic execution:
# → Specification analysis
# → Architecture design
# → TDD implementation
# → Quality validation
```
